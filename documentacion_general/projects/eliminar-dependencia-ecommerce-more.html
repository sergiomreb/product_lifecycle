Eliminar dependencia de ecommerce para lanzar Ecomload

Ir a JIRA → ← Volver

Resumen

Desacoplamiento de Ecomload del módulo de ecommerce, permitiendo su despliegue independiente y reduciendo dependencias innecesarias en el ecosistema de productos.

Actualmente Ecomload tiene acoplamiento fuerte con ecommerce, lo que limita su escalabilidad y hace complicados los despliegues independientes.

Esta iniciativa permitirá que Ecomload sea una plataforma independiente y reutilizable por otros módulos.

Decisiones pendientes

  • Identificar todas las dependencias actuales entre Ecomload y ecommerce
  • Diseñar interfaces de API para integración desacoplada
  • Crear eventos o webhooks para comunicación asíncrona
  • Establecer orden de ejecución de desacoplamiento
  • Validar que otros módulos puedan usar Ecomload independientemente

Temas pendientes completar

  • Auditoría de todas las dependencias ecommerce activas
  • Refactorización de código acoplado
  • Validación de funcionamiento independiente
